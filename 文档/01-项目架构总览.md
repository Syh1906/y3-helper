# 01 - 项目架构总览

## 1. 项目简介

Y3-Helper 是一个 VSCode 扩展，用于辅助开发者使用 Y3 游戏编辑器。它提供初始化项目、启动游戏、调试附加、物编数据管理、Excel 导表、插件系统、ECA 编译、远程终端和 MCP 服务等能力。当前小为版暂停 Y3Maker AI 面板，不再展示右侧 Y3Maker 侧边栏。

---

## 2. 小为版定位

当前仓库是 `Syh1906/y3-helper` 的小为版。

| 项目 | 当前约定 |
|------|----------|
| 仓库地址 | `https://github.com/Syh1906/y3-helper` |
| 扩展身份 | `syh1906.y3-helper-xiaowei` |
| 显示名称 | `Y3开发助手（小为版）` |
| 版本线 | 从 `1.0.0` 开始 |
| 安装方式 | 从源码构建 `.vsix` 后手动安装 |
| 发布范围 | 不发布到 VSCode 插件市场 |
| 自动同步 | 不同步到内网仓库 |

此分叉通过新的 `publisher.name` 与上游公开插件分离。命令 ID、视图 ID 和设置键仍沿用 `y3-helper.*` 与 `Y3-Helper.*`，因为本轮目标是发行身份分离，不是运行时命名空间迁移。

## 3. 项目目录结构

```text
y3-helper/
├── src/                        # TypeScript 源代码
│   ├── extension.ts            # 扩展入口，activate() 函数在此
│   ├── env.ts                  # 环境管理（编辑器路径、地图路径等）
│   ├── config.ts               # 运行时配置
│   ├── debug.ts                # Lua 调试器集成
│   ├── launchGame.ts           # 启动游戏逻辑
│   ├── launchEditor.ts         # 启动编辑器逻辑
│   ├── makeLuaDoc.ts           # 生成 Lua API 文档
│   ├── globalScript.ts         # 全局脚本功能管理
│   ├── networkServer.ts        # 网络服务器（测试 network 库用）
│   ├── mainMenu/               # VSCode 左侧主菜单
│   ├── editorTable/            # 物编数据管理
│   ├── metaBuilder/            # Meta 文件生成器
│   ├── console/                # 游戏与 VSCode 之间的实时通信
│   ├── plugin/                 # 插件系统
│   ├── ecaCompiler/            # ECA 编译为 Lua
│   ├── luaLanguage/            # Lua 语言服务增强
│   ├── customDefine/           # 用户自定义枚举数据
│   ├── mcp/                    # 扩展内置 MCP HTTP 服务
│   ├── codemaker/              # Y3Maker AI 恢复锚点；当前不从 activate() 启动
│   ├── helper_meta/            # 物编字段元数据
│   ├── tools/                  # 基础工具集
│   └── utility/                # 通用工具
├── dist/                       # Webpack 打包输出
│   ├── extension.js            # VSCode 实际加载的扩展入口
│   └── vsix/                   # 本地 VSIX 输出目录
├── out/                        # tsc 编译输出（测试/调试用）
├── template/                   # 模板文件
├── 3rd/                        # 第三方库
├── l10n/                       # 国际化翻译文件
├── image/                      # 扩展图标等图片资源
├── .github/workflows/          # GitHub Actions 构建检查
├── scripts/                    # 本地维护脚本
├── webpack.config.js           # Webpack 打包配置
├── tsconfig.json               # TypeScript 编译配置
├── package.json                # 扩展清单与 npm 依赖
└── 文档/                       # 维护文档目录
```

## 4. 技术栈

| 技术 | 说明 |
|------|------|
| TypeScript | 主要开发语言，编译目标为 ES2022 / Node16 模块 |
| Webpack 5 | 将扩展代码打包为 `dist/extension.js` |
| VSCode Extension API | 提供树形视图、命令、语言服务等 UI 接口 |
| Node.js VM | 在沙箱中运行用户插件 JS 脚本 |
| ExcelJS | 读取 Excel 文件 |
| @vscode/l10n | 国际化运行时 |
| @modelcontextprotocol/sdk | 提供扩展内置 MCP HTTP 服务 |
| GitHub Actions | 只做构建检查，不做发布 |

## 5. 扩展激活条件

扩展通过 `package.json` 中的 `activationEvents` 激活。主要场景包括：

```json
[
  "workspaceContains:header.project",
  "workspaceContains:**/header.project",
  "workspaceContains:y3/更新日志.md",
  "workspaceContains:y3/CHANGELOG.md",
  "onDebugResolve:y3lua",
  "onCommand:y3-helper.startMCPServer",
  "onCommand:y3-helper.stopMCPServer"
]
```

当打开 Y3 地图工程、启动 `y3lua` 调试，或手动执行 MCP 命令时，扩展会被激活。MCP 激活条件仍然有效；暂停的是 Y3Maker AI 面板和 CodeMaker 运行时，不是 `Y3-Helper.MCP.StartMode`。
